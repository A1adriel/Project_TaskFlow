# Project_TaskFlow

## Постановка задачи
Разработать клиентское одностраничное веб-приложение (SPA) — менеджер задач. Приложение должно позволять:
–	добавлять задачи с указанием приоритета (высокий / средний / низкий);
–	отмечать задачи как выполненные;
–	удалять задачи;
–	фильтровать задачи по состоянию (все / активные / выполненные);
–	сохранять данные в localStorage браузера (без серверной части);
–	отображать прогресс выполнения и статистику.
## Выбранная методология и обоснование
Выбрана методология Feature-Driven Development (FDD). Обоснование: задача хорошо разбивается на независимые возможности (features): добавление задач, фильтрация, сохранение, визуализация прогресса. Каждая возможность реализуется и тестируется отдельно. FDD предполагает начало работы с моделью предметной области (список задач как объект), что соответствует паттерну MVC, применённому в проекте.
## Этапы реализации и применённые методы
| Этап | Описание | Применённый метод |
|------|----------|-------------------|
| 1. Анализ требований | Определение функций приложения и структуры данных | Декомпозиция на пользовательские истории (user stories) |
| 2. Проектирование модели | Структура объекта task: id, text, priority, done, createdAt | Абстракция данных — моделирование предметной области |
| 3. Бизнес-логика | Функции addTask, toggleTask, deleteTask, clearDone | Модульное программирование — CRUD изолирован от отображения |
| 4. Представление (View) | Функция render() и генерация HTML по модели данных | Паттерн MVC — разделение данных и представления |
| 5. Контроллер (Events) | Обработчики событий: click, keydown; делегирование | Делегирование событий — один обработчик на список |
| 6. Сохранение данных | Запись/чтение из localStorage при каждом изменении | Метод персистентности — хранение состояния на клиенте |
| 7. Тестирование | Ручное тестирование всех сценариев в браузере | Метод сценарного тестирования (use-case based testing) |

## Архитектура приложения (паттерн MVC)
Приложение построено по паттерну MVC (Model-View-Controller), разделённому на три логических блока:
–	Модель (Model) — массив tasks[] и функции addTask / toggleTask / deleteTask / clearDone. Отвечает за хранение и изменение данных. Состояние сохраняется в localStorage.
–	Представление (View) — функция render(), которая по актуальному состоянию tasks[] генерирует HTML-разметку списка, обновляет счётчики, прогресс-бар и статистику.
–	Контроллер (Controller) — обработчики событий DOM: добавление задачи по кнопке и клавише Enter, делегированный обработчик на список для toggle/delete, кнопки фильтров.

## Ключевые реализации

### Добавление задачи с валидацией входных данных

```javascript
function addTask(text, priority) {
  text = text.trim();          // метод валидации
  if (!text) return false;
  tasks.push({ id: nextId++, text, priority,
    done: false, createdAt: new Date().toLocaleDateString('ru-RU') });
  save();  return true;
}
```

### Функция рендера — единственная точка обновления DOM

```javascript
function render() {
  // 1. Обновить бэджи фильтров
  // 2. Обновить прогресс-бар
  // 3. Отфильтровать и отсортировать задачи
  // 4. Сгенерировать HTML и вставить в DOM
}
```

### Делегирование событий — один обработчик на весь список

```javascript
taskList.addEventListener('click', e => {
  const li = e.target.closest('.task-item');
  const id = parseInt(li.dataset.id, 10);
  if (e.target.closest('.delete-btn')) deleteTask(id);
  else if (e.target.closest('.check-wrap')) toggleTask(id);
  render();
});
```
## Возникшие сложности и способы их решения
| Сложность | Решение |
|-----------|---------|
| XSS-уязвимость при вставке пользовательского текста в innerHTML | Реализована функция `escHtml()`, экранирующая специальные символы HTML |
| Сортировка: активные задачи должны быть выше выполненных | Двухуровневая сортировка: сначала по полю `done`, затем по приоритету |
| Сохранение id при удалении задач (чтобы id не повторялись) | Счётчик `nextId` хранится отдельно в `localStorage` и никогда не обнуляется |
| Адаптивность фильтров (кнопки выходят за экран на малых экранах) | Горизонтальный скролл с `display: flex` + `overflow-x: auto` + скрытый scrollbar |
